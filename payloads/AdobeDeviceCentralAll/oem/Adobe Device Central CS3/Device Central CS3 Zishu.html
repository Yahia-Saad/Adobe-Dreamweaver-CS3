最低系统要求

Device Central 仅可作为 Adobe® CS3 产品的组件安装，或与 Adobe® Creative Suite 3 一起安装。因此，请查看要与 Device Central 一起安装的 Adobe 产品的系统要求。

安装软件

Device Central 仅可作为 Adobe® CS3 产品或 Adobe® Creative Suite 3 的组件安装。
  1. 在安装软件之前, 关闭您计算机上所有正在使用的软件, 包括其他的 Adobe 软件; 微软办公应用软件, 和浏览器窗户.
  2. 请执行以下操作之一：
    • 将 CD 或 DVD 放入驱动器，并按照屏幕上的说明进行操作。（如果安装程序未自动启动，请双击磁盘根级别下的 Setup.exe [在 Windows 环境下] 或 Setup [在 Mac OS 环境下] 以启动安装程序。）
    • 如果您的软件是从 Web 下载的，则请打开文件夹并双击 Setup.exe（在 Windows 环境下）或 Setup（在 Mac OS 环境下），然后按照屏幕上的说明进行操作。

卸载软件

  1. 在卸载软件之前, 关闭您计算机上所有正在使用的软件; 包括其他的 Adobe 软件; 微软办公应用软件, 和浏览器窗户.
  2. 请执行以下操作之一：
    • 在 Windows 环境下，打开 Windows 控制面板，然后双击“添加或删除程序”。 选择要卸载的产品，单击“更改/删除”，然后按照屏幕上的说明进行操作。
    • 在 Mac OS 环境下，双击 Applications\Utilities\Adobe Installers 中的产品安装程序。 作为管理员进行验证，然后选择“卸载组件”并按照屏幕上的说明进行操作。

安装 QuickTime

在 Device Central 中进行视频回放需要在您的系统中安装 Apple QuickTime Player 版本 7 或更高版本。 如果没有安装 QuickTime，则将无法回放 Flash Lite 文档中包含的视频及某些声音格式。 要在您的系统上安装 QuickTime，请转至 Apple 下载站点：http://www.apple.com/quicktime/download，并按照安装说明进行操作。

字体模拟

为了准确模拟 Flash 文档中定义的字体，请确保您的系统安装了需要的字体。 如果 Flash 影片使用 Flash 通用设备字体“类型”（_sans、_serif、_typewriter）中的其中一种，则 Flash Lite 将使用系统上可用的相应字体来替代这些字体。 由于设备上大多数本地字体是制造商专有的，因此您的计算机上可能不具有这些字体。 Device Central“首选项”为三种通用设备字体类型分别提供了普通 Windows 和 Mac OS 系统字体的映射。 如果您已经安装了制造商的实际设备字体，或仅要将其他字体用作默认字体，请转至 Device Central“首选项”，根据需要重新映射通用字体“类型”。 支持其他语言的字体映射。 您可以仅映射系统上安装的字体及符合选定的语言编码的字体。

已知功能

播放器
  • 对于 _capStreamSound 功能，不管设备配置文件信息如何，Flash Lite 1.1 和 Flash Lite 2.x 播放器始终返回“false”。
  • 对于 System.capabilities.hasStreamingAudio 属性，不管设备配置文件信息如何，Flash Lite 2.x 播放器始终返回“true”。
  • 对于 hasQWERTYKeyboard 系统功能，不管设备配置文件信息如何，Flash Lite 2.x 播放器始终返回“true”。

输入

  • 对于某些设备，如果 <mouseType value="Partial"/>，则可能不会正确模拟 mouseType 功能。 这意味着模拟器中支持的鼠标事件与实际设备中支持的鼠标事件可能不同。 请务必验证鼠标事件与在实际设备上的预期一致。
  • 如果 <mouseType value="Stylus"/>，则 hasStylus 系统功能将返回“false”。
  • Device Central 不检查已测试的文件是否符合严格的 BREW 设备文件命名规定。 如果您要将其用于 BREW 设备，请确保文件符合 BREW 规则。

SWF 文件中包括的视频和图像

不管实际设备是否支持 SWF 文件中包括的视频和图像，它们将始终在模拟器中回放。 要检查 SWF 中的视频和图像是否受支持，请选择所需的目标设备，单击“设备配置文件”选项卡，然后转至“Flash”部分。 如果支持 SWF 中的视频和图像，则将列出支持的各种格式。

Web 页中包含的 SWF 内容

在 Windows 环境下，模拟器不回放 Web 页中嵌入的 Flash 内容。 在 Mac OS 环境下，Device Central 使用 Flash 桌面增效工具，这可能会导致 Flash Lite 内容出现意外结果。

我们建议您单独测试嵌入的 Flash Lite 内容。

声音

  • 不回放特定于 BREW 的声音格式（如 QCELP、CMX 和 BCI）。
  • 不管设备配置文件是否表明仅支持单声道声音，soundChannels 功能始终返回为“Stereo”。
  • 对于 multipleDeviceSounds 功能，即使设备配置文件表明不支持多声道混合，但 Flash Lite 1.1 播放器始终返回“Yes”。
  • 如果 SWF 影片具有要同时播放（混合）的嵌入多声道本地 PCM 原声，可能会导致 Flash Lite 2.x 播放器出现一些回放质量问题。
  • 回放 MIDI 文件时可能发生质量问题。 要改善质量，请尝试将 QuickTime 系统设置中的默认合成器更改为“通用 MIDI”。
  • 在 Mac OS 环境下，可能会发生在开始播放时跳过小部分声音的情况。
  • 在 Mac OS 环境下，使用 Flash Lite 2.x 播放器回放本地 MP3、MFI 和 MIDI 格式可能会遇到一些质量问题。
  • 在 Mac Power PC 中，SMAF-5 文件可能会在回放期间跳跃或停止。
  • 快速拖动屏幕上的应用程序窗口时，声音回放可能会不流畅。
性能

每个设备配置文件均包含性能指标信息，即 Adobe 在实际设备运行性能测试得出的结果。 该性能指标使您能够对设备分类。 指标越高，性能越好。 要在计算机上模拟设备性能，必须首先运行校准进程。 在运行校准进程期间，Device Central 将运行在实际设备上进行性能测试期间所使用的 SWF 文件。 然后，您计算机上的校准结果将被设置为与设备性能指标相关联。 请注意，性能模拟是实际设备性能的近似结果。用于校准的 SWF 文件包括文本、位图和矢量渐变。 因此，性能模拟对于这类内容是最准确的。 此性能测试未将某些功能（如 XML 解析、ActionScript 和网络）考虑在内。

由于计算机上运行的其他进程会影响性能，因此要获得最准确的模拟结果，应经常执行设备校准。 必须始终在实际设备上进行最终测试。

FSCommand2 命令

SetFullScreen： 每一个 SWF 文件均按照全屏设置进行模拟。

SetInputTextType： 模拟器忽略对拉丁语/非拉丁语字符的文本输入限制。

即使设备配置文件包含不同的信息，但是系统将假设以下 FSCommand2 命令受支持，然后对其进行模拟。

所有 Flash Lite 播放器：

  • Escape、Unescape
  • GetFreePlayerMemory、GetTotalPlayerMemory
  • GetTimezoneOffset

Flash Lite 2.x 播放器：

  • GetDateDay、GetDateMonth、GetDateWeekday、GetDateYear
  • GetTimeHour、GetTimeMinute、GetTimeSecond
