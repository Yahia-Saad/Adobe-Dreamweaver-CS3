Configuration minimale requise

Vous ne pouvez installer Device Central qu'en tant que composant d'un produit Adobe® CS3 ou avec Adobe® Creative Suite 3. Vous devez donc consulter la configuration requise du produit Adobe pour lequel vous installez Device Central.

Installation du logiciel

Vous ne pouvez installer Device Central qu'en tant que composant d'un produit Adobe® CS3 ou d'Adobe® Creative Suite 3.
  1. Avant de procéder à l'installation du logiciel, fermez toutes les applications, notamment les autres applications Adobe, Microsoft Office et les fenêtres de navigateur.
  2. Procédez selon l'une des méthodes suivantes :
    • Insérez le CD ou le DVD dans le lecteur et suivez les instructions qui s'affichent. Si le programme d'installation ne démarre pas automatiquement, veuillez lancer le processus d'installation en double-cliquant sur le fichier Setup.exe (Windows) ou Setup (Mac OS) au niveau de la racine de votre disque.
    • Si vous avez téléchargé le logiciel depuis Internet, ouvrez le dossier et double-cliquez sur Setup.exe (Windows) ou Setup (Mac OS) puis suivez les instructions qui s'affichent à l'écran.

Désinstallation du logiciel

  1. Avant de procéder à la désinstallation du logiciel, fermez toutes les applications, notamment les autres applications Adobe, Microsoft Office et les fenêtres de navigateur.
  2. Procédez selon l'une des méthodes suivantes :
    • Sous Windows, ouvrez le Panneau de configuration Windows et double-cliquez sur Ajout/suppression de programmes. Sélectionnez le produit que vous souhaitez désinstaller, cliquez sur Modifier/Supprimer puis suivez les instructions qui s'affichent à l'écran.
    • Sous Mac OS, double-cliquez sur le programme d'installation de produit sous Application\Utilitaires\Programme d'installation d'Adobe. Authentifiez-vous en tant qu'administrateur, sélectionnez Désinstaller les composants et suivez les instructions qui s'affichent.

Installation de QuickTime

Pour utiliser la lecture vidéo dans Device Central, vous devez avoir installé sur votre système Apple QuickTime Player version 7. Si vous n'installez pas QuickTime, vous ne pourrez pas visionner les vidéos, ni lire certains formats de sons contenus dans les documents Flash Lite. Pour installer QuickTime sur votre système, rendez-vous sur le site de téléchargement Apple, http://www.apple.com/quicktime/download, et suivez les instructions d'installation.

Emulation des polices

Afin d'émuler correctement les polices définies dans votre document Flash, assurez-vous que la police souhaitée est installée sur votre système. Si votre vidéo Flash utilise un des « types » de polices de périphérique standard (_sans, _serif, _typewriter), Flash Lite le remplacera par la police correspondante disponible dans le système. La plupart des polices d'origine des périphériques étant la propriété du fabricant, vous ne les trouverez peut-être pas sur votre ordinateur. Vous trouverez dans les préférences de Device Central un mappage respectif des polices systèmes courantes des systèmes Windows et Mac pour les trois types de polices de périphériques génériques. Si vous disposez de la police de périphérique du fabricant ou si vous souhaitez juste utiliser une autre police que la police par défaut, allez dans les Préférences de Device Central et remappez les « types » de polices génériques. Le mappage de polices est pris en charge dans différentes langues. Vous ne pouvez mapper que les polices installées sur votre systèmes et qui sont conformes au codage langue sélectionné.

Problèmes connus

Lecteur
  • Les lecteurs Flash Lite 1.1 et Flash Lite 2.x renvoient toujours « false » pour la fonctionnalité _capStreamSound, indépendamment des informations sur le profil du périphérique.
  • Les lecteurs Flash Lite 2.x renvoient toujours « true » pour la propriété System.capabilities.hasStreamingAudio, indépendamment des informations sur le profil du périphérique.
  • Les lecteurs Flash Lite 2.x renvoient toujours « true » pour la fonctionnalité système hasQWERTYKeyboard, indépendamment des informations sur le profil du périphérique.
Entrée
  • Pour certains périphériques, la fonctionnalité mouseType peut ne pas être émulée correctement si <mouseType value="Partial"/>. Les événements de souris pris en charge dans l'émulateur ne sont donc par forcément identiques à ceux pris en charge sur le périphérique actuel. Vérifiez toujours que les événements de souris fonctionnent correctement sur le périphérique réel.
  • La fonctionnalité système hasStylus renvoie « false » si <mouseType value="Stylus"/>.
  • Device Central ne vérifie pas que le fichier testé est conforme aux conventions strictes de dénomination de fichiers périphériques BREW. Si vous souhaitez cibler des périphériques BREW, assurez-vous que votre fichier est conforme aux règles BREW.
Videos et images incluses dans un fichier SWF
Les vidéos et images incluses dans un fichier SWF passent toujours dans l'émulateur, que le périphérique prenne cette fonction en charge ou non. Pour vérifier la prise en charge de vidéos et images dans un fichier SWF, sélectionnez le ou les périphériques cibles, cliquez sur l'onglet Profils de périphérique et allez dans la section Flash. Si les vidéos et images d'un fichier SWF sont prises en charge, vous verrez une liste des formats respectifs.

Contenu SWF intégré dans les pages Web
Sous Windows, l'émulateur ne lit pas le contenu Flash intégré dans les pages Web. Sous Mac, Device Central utilise le module de bureau Flash qui peut donner des résultats inattendus avec un contenu Flash Lite.

Nous vous conseillons de tester séparément les contenus Flash Lite intégrés.

Son
  • Les formats sons propres à BREW, tels que QCELP, CMX et BCI ne sont pas lus.
  • La fonctionnalité soundChannels est toujours « Stereo », même si les profils de périphériques indiquent uniquement une prise en charge des sons monos.
  • Le lecteur Flash Lite 1.1 renvoie toujours « Yes » pour la fonctionnalité multipleDeviceSounds, même si le profil de périphérique indique que le mélange de sons n'est pas pris en charge.
  • Les vidéos SWF avec plusieurs sons bruts PCM natifs intégrés pour une lecture simultanée (mélange) peuvent présenter quelques problèmes de qualité à la lecture sur les lecteurs Flash Lite 2.x.
  • Vous risquez de rencontrer des problèmes de qualité lors de la lecture de fichiers MIDI. Pour une meilleure qualité, allez dans les paramètres système de QuickTime et essayez de modifier le synthétiseur par défaut en MIDI.
  • Sous Mac, il se peut que des parties de sons ne soient pas lues au début.
  • Sous Mac, vous risquez de rencontrer des problèmes de qualité lors de la lecture de fichiers natifs MP3, MFI et MIDI sur les lecteurs Flash Lite 2.x.
  • Sur les ordinateurs Mac Power, les fichiers SMAF-5 risquent de sauter et de s'interrompre lors de la lecture.
  • Le son risque de sauter lorsque vous faites glisser la souris rapidement dans la fenêtre de l'application.
Performance
Chaque périphérique dispose d'informations sur l'indice de performance, déterminé au cours d'un test de performance effectué par Adobe sur le périphérique réel. Cet indice de performance permet de classer les périphériques. Plus il est élevé, plus le périphérique est performant. Avant de pouvoir simuler la performance du périphérique sur votre ordinateur, vous devez exécuter un processus de calibration. Au cours de ce processus, Device Central exécute le fichier SWF utilisé au cours du test de performance sur le périphérique réel. Le résultat de la calibration effectuée sur votre ordinateur est ensuite comparé à l'indice de performance du périphérique.

Remarque : la simulation de performance est une approximation de la véritable performance du périphérique.

Le fichier SWF utilisé pour la calibration contient du texte, des bitmaps et des interpolations vectorielles. La simulation de performance sera donc plus précise pour ce type de contenu. Les fonctionnalités telles que l'analyse XML, ActionScript et la mise en réseau, ne sont pas prises en compte pour ce test de performance.

Pour obtenir des résultats de simulation plus précis, calibrez fréquemment votre périphérique car d'autres processus s'exécutant sur votre ordinateur affectent les performances. Vous devez toujours effectuer le test final sur le périphérique réel.

Commandes FSCommand2 
SetFullScreen : Chaque fichier SWF est émulé comme s'il était en Plein écran.

SetInputTextType : L'émulateur ignore les restrictions de saisie de texte en caractères latin/non latin.

Lorsqu'elles sont prises en charge, les commandes FSCommand2 suivantes sont émulées, même si le profil de périphérique fournit d'autres informations.

Tous les lecteurs Flash Lite :
  • Escape, Unescape 
  • GetFreePlayerMemory, GetTotalPlayerMemory
  • GetTimezoneOffset 
Lecteurs Flash Lite 2.x :
  • GetDateDay, GetDateMonth, GetDateWeekday, GetDateYear
  • GetTimeHour, GetTimeMinute, GetTimeSecond
